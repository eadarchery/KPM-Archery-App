# Dual-Language UI (Bahasa Malaysia + English)

_Added 2026-07-01. Frontend-only. No database changes, no bilingual content columns._

The app UI supports two languages:

- **English (`en`) â€” DEFAULT**
- **Bahasa Malaysia (`ms`) â€” secondary**

> The default was switched from `ms` to `en` on 2026-07-01. Persistence now only
> records an explicit user choice (not the on-load default), and the storage key
> was bumped to `kpm.language.v2` so the change takes effect for users who had
> already auto-saved the old default.

Only **fixed app interface text** is translated. Admin-written content is **never**
translated and renders exactly as stored: article/notification title & body,
school / PLD / state / user names, archer IDs, emails, raw audit action keys, and
uploaded file names.

## How it works

A lightweight, dependency-free system (no i18next):

| File | Role |
|------|------|
| `src/i18n/en.ts` | English dictionary (canonical key shape) |
| `src/i18n/ms.ts` | Bahasa Malaysia dictionary (default) |
| `src/i18n/index.ts` | Types, `translate()` resolver, fallback chain, constants |
| `src/contexts/LanguageContext.tsx` | `LanguageProvider` + `useLanguage()` hook |
| `src/components/layout/LanguageSwitcher.tsx` | Compact **BM / EN** toggle (BM first) |

`LanguageProvider` wraps the app in `src/App.tsx`. The switcher lives in the
`Header` right-hand tools (visible on desktop **and** mobile; it never overflows
the header or the BottomTabBar).

### Usage

```tsx
const { t, language, setLanguage, toggleLanguage } = useLanguage()
t('common.save')                                   // 'Simpan' / 'Save'
t('rolePermissions.resetToDefault', { role: 'Admin 2' })  // {var} interpolation
t('roles.' + role)                                 // dynamic key
```

### Behaviour

- **Default** is `en` (English).
- **Persistence:** an explicit user choice is stored in `localStorage` under
  `kpm.language.v2` and restored on load. The on-load default is **not** persisted,
  so `DEFAULT_LANGUAGE` always governs users who never picked a language.
  `<html lang>` is kept in sync.
- **Missing-key fallback:** current language â†’ `en` â†’ the key string. The page never
  crashes on a missing key.
- Keys are **dot-paths** into nested dictionaries (`common`, `nav`, `roles`,
  `status`, `menu`, `access`, `feature`, `comingSoon`, `maintenance`, `systemRules`,
  `rolePermissions`, `roleOverview`, `superAdmin`, `articles`, `notifications`,
  `achievements`, `scores`, `reports`).
